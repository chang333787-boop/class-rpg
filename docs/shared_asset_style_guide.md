# 공유 에셋 스타일 가이드 — class-rpg ↔ 별빛 아레나

> 목적: **우리반 성장 RPG(class-rpg)** 와 **별빛 아레나(star-arena)** 가 나중에 합쳐지거나 나란히 놓여도
> **한 세계처럼 보이도록**, 두 게임의 새 이미지 에셋이 따르는 **공통 시각 규칙**을 정한다.
>
> 상태: **초안(2026-07-13, Claude Code).** 아래 §6 "열린 결정"은 사용자 확정 필요.
> 이 문서는 두 repo가 공유한다 — 확정 후 `star-arena` repo에도 같은 파일을 두어 동기화한다.
>
> ⚠️ 이건 **시각 스타일**만 통일한다. 저장 방식·계정·게임 로직·Firebase 규칙은 각 게임이 그대로 유지한다
> (그쪽은 두 게임의 성격이 달라 통일 대상 아님 — 각 repo의 안전 규칙이 우선).

---

## 0. 왜 이 문서가 필요한가

두 게임의 아트는 이미 **같은 계열**이다(아래 §1). 하지만 앞으로 새 에셋을 각각 만들다 보면 미묘하게
갈라진다. 합칠 가능성을 염두에 두면, 지금부터 **같은 시각 언어**를 문서로 못박아 두는 게 싸게 먹힌다.
기존 에셋을 다시 그리는 게 아니라, **앞으로 만드는 에셋의 기준**을 세우는 문서다.

---

## 1. 공통 시각 DNA (두 게임 이미 공유 중 — 관찰 근거)

class-rpg(몬스터·씨앗·작물·장비·스킬북)와 별빛 아레나(몬스터·캐릭터·바위·덤불) 실제 에셋을 비교한 공통점:

- **밝고 채도 높은 색** — 파스텔이 아니라 또렷하고 명랑한 색.
- **굵고 짙은 외곽선** — 개체 윤곽을 분명한 선으로 감싼다.
- **부드러운 셀 셰이딩** — 광원은 **위쪽 하나**. 그라데이션은 단순한 2~3단.
- **반짝이/별 하이라이트** — 두 게임의 시그니처. 젤리·보석·표지에 작은 ✦ 반짝임.
- **둥글고 친근한 형태** — 각지지 않게, 말랑하고 통통하게.
- **투명 배경(알파)** — 항상. 바닥 그림자·액자·배경 없음.
- **초등 친화·귀여움** — 무섭거나 사실적이지 않게.

→ 이 7가지는 **양쪽 새 에셋 모두 지킨다.** 여기까진 이미 합의된 상태다.

---

## 2. ⚠️ 관찰된 차이 — 디테일 수준 (결정 필요, §6-A)

같은 DNA지만 **정교함의 정도**가 다르다:

| | class-rpg | 별빛 아레나 |
|---|---|---|
| 인상 | 디테일 많은 "프리미엄 RPG 아이콘" | 단순·귀여운 "카와이 블롭" |
| 외곽선 | 굵고 진함 | 얇고 부드러움 |
| 음영·장식 | 풍부(금테·보석·질감) | 미니멀(면 위주) |
| 예 | 왕관·무지개검·마법 교본 | soft_jelly·학생 캐릭터·보스 슬라임 |

합쳐서 통일하려면 **목표 디테일 수준**을 정해야 한다(§6-A). 정하기 전까지는 각자 유지.

---

## 3. 공통 팔레트 (권장)

두 게임이 색을 공유하면 세계관이 하나로 묶인다.

- **속성색**: 불 `#FF6B35~#FF3D00`(빨강·주황) / 물 `#4FC3F7~#1976D2`(파랑·하늘) / 풀 `#7CB342~#33691E`(초록·연두).
- **등급·희귀도 강조**: 금색 `#FFD700~#FF8F00`(전설/최고 티어) + 보석색(빨강/파랑/초록 다이아).
- **중립/기본**: 은색·회색(`#BDBDBD~#607D8B`).
- 개체 하나 안에서 주색 1~2 + 강조색(금/반짝) 조합. 너무 많은 색 금지.

*(위 hex는 class-rpg equipment 팔레트에서 뽑은 출발점 — 별빛 값과 대조 후 §6에서 확정.)*

---

## 4. 콘텐츠·톤 정책

**공통(양쪽 지킴):**
- 유명 게임/애니 캐릭터를 그대로 따라 그리지 않는다(마리오·포켓몬 X). 학생 창작 존중.
- 초등 적합 — 잔인·공포·선정 요소 없음.
- 글자·숫자·워터마크를 이미지 안에 넣지 않는다(아이콘은 그림만).

**⚠️ 차이(결정 필요, §6-B):**
- **별빛 아레나**는 "놀이도구" 지향 — 총·칼·폭탄 대신 별조각·비눗방울·종이비행기, 화면 용어도 "발사/에너지/리턴".
- **class-rpg**는 RPG라 **검·지팡이·몬스터 전투**가 이미 있다(장비 e_w 무기 등).
- 합칠 때 톤을 어느 쪽으로 맞출지 = 사용자 결정.

---

## 5. 폴백 철학 (공통 — 양쪽 이미 동일)

**PNG가 없어도 게임은 절대 멈추지 않는다.** 이미지가 없거나 이름이 틀리면 자동으로 대체 표시:
- class-rpg: **이모지 폴백**(`iconImg`의 `onerror` → 이모지 span).
- 별빛 아레나: **도형 폴백**(원형 탄환·도형 캐릭터, `ASSETS_ENABLED`/F2로 확인).

새 에셋을 추가할 때도 이 폴백을 깨지 않는다(원 데이터의 이모지/도형 유지).

---

## 6. 기술 규격은 게임별로 다름 (통일 대상 아님)

렌더 방식이 달라서 크기·폴더·상태 수는 각 게임 고유다. **시각 스타일만 공유, 규격은 각자.**

| | class-rpg | 별빛 아레나 |
|---|---|---|
| 렌더 | DOM `<img>` (CSS 축소) | Canvas `drawImage` (실시간) |
| 크기 | 512×512 | 128×128(캐릭터) 등 |
| 폴더 | flat `assets/<kind>/<id>.png` | `assets/<cat>/<id>/<state>.png` (상태별 다장) |
| 상태 | 정적 1장 | idle/move/attack/hit/return 등 다프레임 |
| 스위치 | 항상 켜짐(경로 규약) | `ASSETS_ENABLED` 플래그 |
| 상세 규격 | `docs/asset_implementation_spec.md` §2 | `ASSET_GUIDE.md` / `ASSET_SLICE_FIX_NOTE.md` |

## 7. 열린 결정 (사용자 확정 필요)

- **A. 목표 디테일 수준** — ① 별빛식으로 더 단순·귀엽게 통일 / ② class-rpg식 디테일로 / ③ 정의된 중간 지점.
  (기존 에셋 재작업 여부도 함께: 앞으로만 적용 vs 점진 교체.)
- **B. 콘텐츠 톤** — 합쳐진 세계에서 class-rpg의 검·전투를 유지할지, 별빛식 "놀이도구" 톤으로 맞출지.
- **C. 공통 팔레트 hex 확정** — §3 값을 별빛 실제 색과 대조해 최종 확정.
- **D. 문서 위치** — 확정본을 `star-arena` repo에도 복사해 양쪽 동기화할지(권장).
